# System Overview

FluentPath English is an English learning platform with a personalized AI Teacher for every student.

## Public Experience

Visitors see:
- Product landing page explaining what FluentPath is
- Course catalog
- Login and signup pages

## Student Experience

After login, a student sees:
- **Student dashboard** — AI greeting, fluency/listening/pronunciation meters, recommended practice
- **Placement test** — 7 questions generated by Claude (grammar, vocabulary, reading, listening) evaluated server-side; result stored in PostgreSQL and shown across all devices
- **AI baseline** — feedback and practice priorities from Claude, based on test performance
- **Courses and lessons** — course catalog; lesson content in progress
- **Pronunciation practice** — phrase drill with local recording
- **In-person class recording** — local audio/video with consent gate
- **Account settings** — profile, address, password

Students who sign up without a teacher invite start with `assignment_status = pending_assignment` until assigned by an admin.

## Teacher Experience

After login, a teacher sees:
- **Teacher dashboard** — assigned student list with progress, difficulty, and next action
- **AI Level Review panel** — suggestions from the Placement Agent when a student's real level differs from self-reported; teacher can approve (updates the level) or dismiss
- **Invite link** — shareable URL that auto-assigns new signups to this teacher

## Admin Experience

Administrators see:
- **School metrics** — student/teacher counts, revenue summary, progress overview
- **AI Level Review panel** — all pending level suggestions across all students (including unassigned)
- **Student and teacher management** — create and edit accounts
- **Assignment management** — assign students to teachers
- **Plan and course management** — create and edit plans and courses

## AI Layers — Current State

### Placement Agent (implemented)
- Runs after every student signup using the motivation text as a writing sample
- Student can retake at any time from the dashboard
- Generates 7 questions via `GET /api/placement/questions` using Claude Haiku with the full assessment KB as system prompt (grammar, vocabulary, reading, listening guides)
- Evaluates answers via `POST /api/placement` and returns: feedback, level estimate, practice priorities
- If the assessed level differs from self-reported, a level suggestion is queued for teacher/admin review

### AI Coach (placeholder)
- Dashboard greeting and summary are static copy personalized from the student profile
- Not yet connected to Claude API

### AI Teacher (placeholder)

## Core Architecture

```
Browser (SPA)
    │
    │  HTTP (JSON API)
    ▼
Node.js HTTP Server
    ├── src/server/routes/     — auth, placement, account, teacher, admin, pronunciation
    ├── src/agents/            — Claude API calls (placement implemented)
    ├── src/storage/           — PostgresStorage / MemoryStorage
    └── src/security.js        — scrypt, tokens
    │
    ▼
PostgreSQL
    └── Users, profiles, sessions, assignments, invites,
        ai_feedback (placement + level suggestions),
        lesson_progress, pronunciation_attempts,
        courses, lessons, plans, payments, consent/media records
```

## Privacy Principles

- Student-specific memory is isolated per student — never mixed into the global KB
- Teacher access to student records is assignment-based (`teacher_student_assignments`)
- Raw media is not processed until consent, retention, and deletion workflows exist
- Passwords hashed with scrypt, timing-safe comparison
