# System Overview

FluentPath English is an English learning platform with a personalized AI Teacher for every student.

## Student Experience

After login, a student sees:

- **Student dashboard** — AI greeting, fluency/listening/pronunciation meters, notification panel for admin messages
- **Placement test** — 8 questions generated by Claude (grammar, vocabulary, reading, listening, speaking); listening uses ElevenLabs TTS with two natural voices; scored 0-100%; stored in PostgreSQL across all devices
- **AI baseline** — feedback, score, level estimate, and practice priorities from Claude
- **My Tests** — compact list of past tests (date, level, score); tap to view detail with questions, answers, AI feedback, and priorities
- **Courses and lessons** — course catalog; lesson content not yet built
- **Pronunciation practice** — phrase drill with local recording
- **In-person class recording** — record in browser or upload file (iOS .m4a supported); AssemblyAI transcribes with speaker diarization; Claude analyzes lesson content
- **Account settings** — profile, address, password

## Teacher Experience

After login, a teacher sees:

- **Teacher dashboard** — assigned students with progress, difficulty, next action, and "View tests →" link
- **AI Level Review panel** — approve or dismiss level suggestions from placement test
- **Student test history** — all placement tests per student with scores, questions, and answers
- **Invite link** — shareable URL that auto-assigns new signups to this teacher

## Admin Experience

After login, an admin sees:

- **Overview (`#admin`)** — clickable metric cards linking to sub-pages, quick links, AI level review, activity
- **Students (`#admin-students`)** — register/edit, progress table with "📤 Send" button, search/filter by name/email/level/goal/teacher, 10 per page
- **Teachers (`#admin-teachers`)** — register/edit, search/filter by name/specialty/status, 10 per page
- **Assignments (`#admin-assignments`)** — assign students to teachers, view all assignments, filter by student/teacher/source, 10 per page
- **Plans (`#admin-plans`)** — manage pricing plans, 10 per page
- **Courses (`#admin-courses`)** — manage course catalog, 10 per page
- **Send recommendation** — admin clicks "📤 Send" next to a student's AI recommendation → student sees a yellow notification on their dashboard

## AI Layers — Current State

### Placement Agent (implemented — `src/agents/placement.js`)
- Auto-runs after signup using motivation text
- Student retakes at any time from dashboard
- `GET /api/placement/questions` — Claude generates 8 questions; listening uses ElevenLabs TTS
- `POST /api/placement` — Claude evaluates, returns `{ feedback, level, score, metrics, priorities }`
- If assessed level ≠ self-reported → level suggestion queued for teacher/admin review
- Questions and answers saved in `ai_feedback.source_data` (JSON) for full test history

### Lesson Analysis Agent (implemented — `src/agents/lesson-analysis.js`)
- `POST /api/recordings` — student uploads class audio
- AssemblyAI transcribes with speaker diarization (`speech_models: ["universal-2"]`)
- Claude analyzes transcript → mistakes, vocabulary, recommendations
- Result shown in LESSON ANALYSIS panel on dashboard

### AI Coach (placeholder)
- Dashboard greeting is static copy personalized from student profile
- Not connected to Claude API

### AI Teacher (placeholder)
- Spec defined in `agents/ai-teacher.md`
- Not yet wired to Claude API

### Pronunciation Agent (placeholder)
- Attempt metadata stored in `pronunciation_attempts`
- Audio upload and scoring not yet built

## Core Architecture

```
Browser (SPA — app.js)
    │  HTTP / JSON API
    ▼
Node.js HTTP Server (server.js)
    ├── src/server/routes/     — auth, placement, account, notifications,
    │                            teacher, admin, recordings, tts, pronunciation
    ├── src/agents/            — Claude API (placement) + AssemblyAI + Claude (lesson)
    ├── src/storage/           — PostgresStorage / MemoryStorage
    └── src/security.js        — scrypt, tokens
    │
    ▼
PostgreSQL
    ├── users, sessions, student/teacher/admin profiles
    ├── teacher_student_assignments, teacher_invites
    ├── ai_feedback (source_type: placement / level_suggestion / lesson_recording)
    ├── lesson_recordings (path, transcript, processing_status)
    ├── student_notifications (message, read_at)
    ├── lesson_progress, pronunciation_attempts
    └── courses, lessons, plans, payments, consent/media records

External APIs
    ├── Anthropic Claude Haiku  — placement + lesson analysis
    ├── AssemblyAI              — class recording transcription
    └── ElevenLabs              — listening dialogue TTS
```

## Privacy Principles

- Student-specific memory is isolated per student — never mixed into global KB
- Teacher access to student records is assignment-based (`teacher_student_assignments`)
- Class recording analysis is gated by explicit UI consent checkbox
- Passwords hashed with scrypt, timing-safe comparison
- Retention and deletion workflows for media are not yet hardened
